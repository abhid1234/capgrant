A fleet of AI coding agents runs with far too much authority — each can write anywhere, run anything, reach any endpoint the process can. capgrant is the missing authorization layer: an issuer hands an agent a scoped, expiring capability grant — [ fs.write : src/** ] for the next 30 minutes — and before it acts, the agent checks the gate. Grants delegate into strict subsets and revoke in a cascade. Everything below runs the real, unmodified library client-side.

v0.3 — human in the loop: a grant answers “what is this agent pre-authorized to do.” But when an action isn't covered by any live grant, the agent doesn't just hard-fail — it raises an approval_request to a human. The real check() returns a soft needs_approval, not a flat DENY. A human approves → the real resolveRecords fold mints a just-in-time grant — scoped to exactly that action, expiring on a TTL, parented to the request — and the same check() now returns ALLOW. Deny and it stays denied. Grant ahead of time, or request → approve just in time.

Quickstart — adopt it in a minute

Three ways a fleet uses it.

A CLI for one-off grants and checks, a library to gate actions inside your own agent runtime, or a git pre-commit hook that checks every staged write against the committing agent's grants.

① CLI & check gate

Issue a scoped grant, then gate an action — exit 0 allowed, 1 denied. No install needed.

# no install needed
npx @avee1234/capgrant grant \
  --subject builder \
  --cap "fs.write:src/**" --ttl 30m

npx @avee1234/capgrant check \
  fs.write src/auth/login.ts \
  --subject builder

② Library

Check before the agent acts, inside your own runtime. Zero dependencies, pure ESM.

import { check } from
  '@avee1234/capgrant';

// gate the action
const r = check(action, resource,
  grants, { subject, now });
if (!r.allowed) refuse(r.reason);

③ Git hook

A pre-commit hook checks each staged write against the agent's grants. Advisory by default.

capgrant hook install

# enforce: out-of-scope
# write aborts the commit
CAPGRANT_ENFORCE=1 \
  git commit -m "…"

What's in the box

Six verbs, one format, a git hook.

capgrant grant

Grant

Issue a scoped, expiring capability grant — action:resource with a TTL — validated before it's written.

capgrant check

Check

The pre-action gate: does a live grant for this subject cover the request? Exit 0 allowed, 1 denied, specific reason.

capgrant delegate

Delegate

Mint a strictly narrower sub-grant from a delegable parent. Every capability must fall inside the parent's authority.

capgrant revoke

Revoke

Withdraw a grant by appending a revocation — and cascade to every grant delegated from it, to a fixed point.

capgrant audit

Audit

Replay a batch of actions and score whether the fleet stayed in scope, each timestamped against the grant that was live then.

capgrant hook

Git hook

A pre-commit hook that treats a commit as a batch of fs.writes and checks each staged path. Advisory by default.
